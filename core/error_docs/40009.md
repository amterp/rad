# RAD40009: Duplicate Parameter

A function or lambda has two parameters with the same name. Parameter
names must be unique within a single parameter list so the body can
refer to each one unambiguously.

## Example

```rad
// This is an error: 'x' is declared twice.
fn add(x, x):
    return x + x
```

## Fix

Rename one of the parameters so each binding has a distinct name:

```rad
fn add(a, b):
    return a + b
```

Shadowing an outer-scope name with a parameter is fine - that's a
common pattern. This error only fires when two parameters of the
same function or lambda collide with each other.
