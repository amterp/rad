# RAD30001: Type Mismatch

A value of one type was used where a different type was expected. Rad is
dynamically typed but still enforces type compatibility in certain contexts.

## Common Causes

1. **Function argument types** - Passing a string to a function expecting an int
2. **Return type annotations** - Returning a value that doesn't match the declared return type
3. **Operator incompatibility** - Using operators with incompatible types

## Examples

```rad
# Function expects int, got string
fn double(n int) -> int:
    return n * 2

double("hello")  # Error: type mismatch
```

```rad
# Return type mismatch
fn get_count() -> int:
    return "not a number"  # Error: expected int, got str
```

## How to Fix

1. **Check the function signature** - Look at what types the function expects
2. **Use type conversion** - Convert values to the expected type:
   ```rad
   x = "42"
   double(int(x))  # Convert string to int first
   ```
3. **Fix the return type** - Either change the return value or the type annotation

## Type Conversion Functions

- `int(value)` - Convert to integer
- `float(value)` - Convert to float
- `str(value)` - Convert to string
- `bool(value)` - Convert to boolean
